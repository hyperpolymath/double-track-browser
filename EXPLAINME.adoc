== Key Claims From README

[quote, README section "Core Architecture"]
____
This project is built on a hybrid architecture that prioritizes both performance and memory safety:

- *Rust Core*: Powers the profile generation and activity simulation engine, compiled to WebAssembly for browser integration
- *ReScript*: Type-safe browser API integration and UI components (compiles to JavaScript)
- *Deno*: Modern runtime for build scripts and development tooling
- *WebExtensions API*: Provides the hooks needed for background operation
____

*Evidence:* The project structure (README line 36-50) shows functional implementation:
- `rust_core/` — WASM-compiled Rust profile generator and activity simulator
- `src/` — ReScript source with background service worker, bindings, content script, dashboard, options page, popup, types, utils
- `scripts/` — Deno build scripts
- `rescript.json` + `deno.json` — Build configuration
- `justfile` — Task runner for build, test, lint

The RSR compliance audit (lines 88-100) confirms zero unsafe blocks and comprehensive testing coverage.

*Caveat:* The project is explicitly experimental (line 87) and warns about unintended consequences. The privacy approach (visibility as camouflage) is novel and unproven at scale.

[quote, README section "Why DoubleTrack Exists"]
____
DoubleTrack explores a different hypothesis: What if, instead of trying to be invisible, you become deliberately, strategically visible—but in ways that obscure rather than reveal?
____

*Evidence:* The ecosystem integration (lines 171-229) connects three sibling projects that provide the theoretical and practical foundation:
1. **Defensive Multiplicity** — ethical framework (purpose-bound identities, 180-day max lifespan, cryptographic accountability)
2. **Consent-Aware HTTP** — web standards (AIBDP manifests, HTTP 430 respect, consent caching)
3. **MAA Framework** — formal verification (Idris2 ABI proofs, Zig FFI identity isolation)

This is more than experimental code — it's grounded in formal ethics (Defensive Multiplicity), internet standards (AIBDP Internet-Draft), and formal verification (Idris2 proofs).

*Caveat:* Integration targets (marked as "Planned" with `[ ]`) are not yet implemented. The core extension works; ecosystem bridges are in design stage.
