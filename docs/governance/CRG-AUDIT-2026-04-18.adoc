=== CATASTROPHIC FINDING: TypeScript used for the entire test suite

The ambient `~/.claude/CLAUDE.md` and the project's own
`.claude/CLAUDE.md` both state:

[quote, Hyperpolymath Language Policy]
____
**BANNED - Do Not Use** … TypeScript → ReScript … Node.js → Deno …
**No new TypeScript files** - Convert existing TS to ReScript.
____

Five `.ts` files are committed to this repository:

[cols="2,1,3"]
|===
| File | Purpose | Status under policy

| `tests/unit/types_test.ts`        | 4 Deno.test cases (unit)     | *policy violation*
| `tests/property/extension_properties_test.ts` | 9 Deno.test cases (property) | *policy violation*
| `tests/e2e/extension_lifecycle_test.ts`       | 9 Deno.test cases (e2e)      | *policy violation*
| `tests/aspect/security_test.ts`               | 10 Deno.test cases (aspect)  | *policy violation*
| `scripts/build.ts`                            | Deno build orchestrator       | *policy violation*
|===

The repo *ships its own `ts-blocker.yml` workflow* (which checks newly
added `.ts` files in PR diffs) and a `validate-rsr` recipe in the
Justfile that prints "✅ No tsconfig.json (ReScript enforced)". Both
are satisfied on technicalities:

- `ts-blocker.yml` fires on `git diff HEAD~1` only, so pre-existing
  `.ts` files are grandfathered and never flagged.
- No `tsconfig.json` exists, but `deno.json` carries a
  `compilerOptions.strict = true` block and `lib: ["deno.window", "dom"]`
  — Deno is the one permitted TS executor in the estate, but the
  policy still says "No new TypeScript files", not "Deno-run TS is
  fine". The `.ts` test suite *is* new TypeScript code written after
  the ReScript-preferred policy was in force.

The violation is compounded by the Justfile:

[source,bash]
----
npx rescript build           # line 35
npx rescript build 2>&1 …    # line 142
npx rescript build -w        # line 174
----

`npx` is part of the npm toolchain, which is banned (Enforcement Rule
2-3). Invoking `npx` to run `rescript` pulls the npm world back in
through the side door, and violates the "No package.json for runtime
deps" / "No node_modules in production" enforcement rules the moment
anyone actually executes these recipes. (`rescript` is available via
Deno equivalents and via direct-install; `npx` is not the right
invocation.)

This single finding blocks *two* CRG levels simultaneously:

. **D-floor "RSR compliance mandatory"** fails: the estate language
  policy is part of RSR, and the repo contains non-trivial source
  files in a banned language (TypeScript) plus three invocations of a
  banned package manager (`npx`).
. **C "deep per-file annotation"** is moot while D-floor is failing.

It also breaks the "honest D > dishonest B" clause: a self-declared
C + `completion-percentage = 60` coexisting with 5 policy-violating
`.ts` files (four of which are *test files*, precisely the artefacts
C-grade evidence rests on) is the kind of honesty gap v2.0 STRICT was
designed to surface.

The fix is mechanical — rewrite the four Deno test files as
`*.test.res` + generated `*.res.js` runnable under `deno test`, and
drop `scripts/build.ts` in favour of a ReScript or Rust build
orchestrator; but until that work lands, the audit cannot certify D.

== Verdict

*Current CRG grade: **E (Pre-alpha)**. Demote two grades from
self-declared C.*

Self-declared: *C*. Audited: *E*. Same two-grade delta as
neurophone's audit earlier today — but for a structurally cleaner
reason: the underlying Rust code is genuinely good (2213 LOC across 6
modules, 54 tests, zero `unsafe`, zero dangerous patterns, proper
proptest integration, well-split aspect/e2e/property/integration test
files). What blocks it is *language policy*, not *code quality*.

Reasoning (one sentence): the repo declares itself a
ReScript+Rust+Deno stack but authors 37% of its tests plus its build
script in TypeScript — a hyperpolymath-banned language whose
presence blocks the "RSR compliance mandatory" D-floor clause
outright, regardless of how clean the remaining 63% of tests are.

A D grade would require removing the 5 `.ts` files (rewrite as
`*.res`) and replacing `npx rescript` with a non-npm-backed
invocation. A C grade would require those *plus* per-directory
READMEs, a populated `[dogfooding-status]`, and a `docs/` annotation
surface comparable to boj-server or gossamer.

The E grade is earned, not conservative-by-default, on four positive
signals that keep it above X/F:

1. Real Rust code: 6 non-stub modules totalling 2 213 LOC with
   genuine content (interests.rs 1004 LOC is the heaviest — likely
   the interest-generation engine). No stub crates.
2. Zero dangerous-pattern uses (`believe_me`, `assert_total`,
   `Admitted`, `sorry`, `unsafeCoerce`, `Obj.magic`) and zero
   `unsafe` blocks in Rust.
3. 54 live Rust `#[test]` / `#[tokio::test]` attributes split
   thoughtfully across inline + integration + property + e2e +
   aspect/security suites — the test *architecture* is C-grade.
4. Contractile 6-verb layout is the most complete seen in audits
   this session (`intend`/`trust`/`must`/`lust`/`dust`/`k9` dirs plus
   top-level `ADJUST.contractile`).

What blocks *immediate* promotion to D (do these first):

. **Rewrite the 4 Deno test files as ReScript** — `tests/unit`,
  `tests/property`, `tests/e2e`, `tests/aspect`. The Rust side of
  each suite is already in `rust_core/tests/`; the `.ts` halves are
  interface-level assertions on message shape / storage schema /
  form data / activity log / extension lifecycle / security aspects,
  all of which ReScript handles idiomatically. 32 Deno.test cases
  → 32 ReScript test cases, one-for-one.
. **Rewrite `scripts/build.ts` as ReScript or Rust.** It's a build
  orchestrator (`deno run --allow-read --allow-write`); either
  language covers it.
. **Drop `npx rescript build` from the Justfile.** Options: install
  `rescript` standalone, use `deno run npm:rescript@11` (explicit
  Deno-npm shim — acceptable under Deno runtime policy), or invoke
  through a Deno task that avoids the `npx` prefix.
. **Fix Justfile SPDX line** — replace
  `# SPDX-License-Identifier: MIT OR Palimpsest-0.8` with
  `# SPDX-License-Identifier: PMPL-1.0-or-later` per Rule 1.
. **Add `[blockers]` section to STATE.a2ml** citing this audit.

What blocks promotion to C (do after D):

. Add `[dogfooding-status]` to STATE.a2ml with concrete done-entries:
  "installed in my daily-driver browser on `<date>`, `<N>` days of
  background activity logged, `<link-to-log>`".
. Per-directory READMEs for at least 10 of the 12 top-level source
  subdirectories. `rust_core/`, `src/`, and `tests/` are non-optional.
. Build out `docs/` with `architecture/`, `decisions/`, `governance/`,
  `practice/` subtrees. Start with an ADR explaining the
  Rust+WASM+ReScript+Deno choice.
. Either implement the three ecosystem integrations (Defensive
  Multiplicity, Consent-Aware HTTP, MAA) or drop them from EXPLAINME's
  "Dogfooded Across The Account" section. Planned integrations are
  not dogfood evidence.
. Cite a CI-green SHA in STATE.a2ml.
. Expand STATE.a2ml from 44 lines to include `[crates]` /
  `[modules]`, `[route-to-mvp]`, `[session-history]` sections (see
  neurophone / gossamer STATE files for shape).

What blocks promotion to B (after C):

. Zero external targets. Need 6+ diverse — different browsers
  (Firefox, Chromium, Brave, Safari-if-possible), different AIBDP
  host sites, different ecosystem integration downstream users.
. No external-contributor flow. Dependabot PRs don't count.
. `RSR-Gold 91%` README badge should be either externally verified
  (OpenSSF Scorecard, third-party audit) or downgraded to "self-audit
  91% — see RSR_COMPLIANCE_AUDIT.md" with a date.

== Notes on Non-Negotiables Respected by this Audit

- The language-policy finding is treated as the dominant blocker
  because both the ambient `~/.claude/CLAUDE.md` and the repo's own
  `.claude/CLAUDE.md` state "No new TypeScript files" as an
  **Enforcement Rule** — not a suggestion. RSR-Gold compliance at
  91% cannot be claimed while the "Gold" rule-set itself is
  violated.
- The `.ts` files are *Deno-executed*, which is the one permitted
  context for the TypeScript *language* in the estate (MCP glue,
  Deno APIs). However, *new* TypeScript code written *for any
  reason* is banned. These test files and this build script could
  have been written in ReScript; they weren't. That's the policy
  gap.
- The repo *ships* a `ts-blocker.yml` CI workflow — a strong signal
  that the project knows the policy. The workflow's
  `HEAD~1`-scoped diff check lets the pre-existing `.ts` files slip
  through. The policy intent is clear; the enforcement is
  grandfathered.
- The self-declared `RSR-Gold%20Level%20(91%25)` badge is *not*
  re-scored by this audit. CRG is a different axis from RSR-Gold;
  both axes should be self-consistent, but the audit only grades
  CRG.
- `completion-percentage = 60` is roughly consistent with the
  evidence (core logic + bindings built, ecosystem integrations at
  0%). This audit does not re-score completion.
